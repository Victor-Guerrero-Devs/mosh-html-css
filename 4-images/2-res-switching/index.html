Resolution Switching

Problem

IN the last lesson, we learned how we can support high density screens by using images with higher pixel counts, so the image won't look elongated and pixelated on screens with high DPR.

There is another problem, on top of providing high res images for screens with high DPR, we also have to provide different image sizes for different devices, i.e. a 300x300 image for mobile, 600x600 for tablets, 1200x1200 for desktops.

In the last lesson, no matter the resolution of the image, it was limited to 400 pixels in width. This time, we want to change the actual width so the image becomes physically smaller or larger depending on the device.

Since there way too many devices and differences between them, we should only provide three kinds of images for resolution switching: small for mobile, medium for tablet, and large for desktop.

How To Implement

Like in the last section, we use the srcset attribute in the image element but with a slight modification.


Instead of using 1x, 2x, 3x, we give the actual number for its width in pixels, hence 400w, 800w, 1200w. Be careful though since you may use "px" instead of "w".

The browser will know which image to use in relation to the device's viewport. Therefore, if the user is visiting your site on a mobile device, the browser will know to pick the image that has 400w

Notice how the image has the class cow? Well, instead of limiting the width of the image to 400px, we make sure it fills its container by using .cow { width: 100%; } in this way, the image becomes responsive.

This is the most basic way to use resolution switching.

Advanced Usage of Resolution Switching

So far, we have implemented basic resolution switching. IN other words, our image takes up 100% of its container's width no matter if it is on mobile or tablet or desktop, while changing physical size in relation to device.

Advanced resolution switching, controls how much of the viewport width it takes up, e.g. 100% for mobile, 50% for tablet, 33% for desktop.

IN order to do this, we get rid .cow { width: 100%; } and use the attribute called sizes as so:

To reword what sizes is doing: if the viewport width is maximum 500px (i.e. mobile), have the image take up 100% of the vw.

Else, if the viewport width is maximum 700px (i.e. tablet), have the image take up 50% of the vw.

Otherwise, have the image take up 33% of the vw.

Conclusion

The numbers used in both the srcset and sizes attributes are completly arbitrary. You can play around with them and use more or less than 3 different sizes.

You should check out Responsive Breakpoints to have this website do all of the math for you.

A couple of tips for using this website:

  • Resolution 200 - 1400 covers the majority of devices
  • Size step determines the difference in file size per image (50kb is ideal)
  • Reduce maximum images to 3 or 5. 20 is way too much.
